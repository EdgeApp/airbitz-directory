Airbitz Directory API

  Airbitz Directory API

Airbitz accepts directory submissions via a RESTful api. Below you will find examples and capabilities for each REST endpoint.

Business Submit

api/v1/business/submit/

Use when adding a new business to Airbitz directory.

api/v1/business/submit/<provider_id>

Use when updating a previously submitted business.

The following is a table of the required and optional parameters for submitting businesses to the Airbitz directory.

Param Required Type Description / Example
provider_id true String ID for your reference when querying or updating records
name true String Business name
provider_url false String URL of listing on external directory to be linked
description false String Description of business
website false String Website url of the business
phone false String Phone number of business
street_address false String Street address of business
neighborhood false String Friendly name or nickname for area
admin3_name false String City name of business
admin2_name false String County name of business
admin1_code false String US FIPS 5-2 1st level administrative division code (e.g., state/province)
postalcode false String Postal code of business
country false String ISO 3166-1 ISO 3166-1 alpha-2 country code
physical_business false Boolean Does the business accept bitcoin at brick and mortar locations
online_business false Boolean Does the business accept bitcoin online
bitcoin_discount false Integer Percent discount given for bitcoin purchases
score false Integer Relative score for popularity or bitcoin transaction volume
location false Object
Param Required Type Description
latitude false Integer Latitude coordinate of the area centroid
logitude false Integer Longitude coordinate of the area centroid
categories false List Business category
expense_category false String Default expense category to tag transactions with
images false List of imgObjects
